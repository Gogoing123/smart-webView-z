1・NPM：node pakeger manager 是   node 包管理器，作用是管理node包（node包可以理解为工具/插件/模块/一段代码）管理的操作：下载安装，查看，卸载，搜索，更新，发布等等

npm在新版本nodejs中已经集成，不需要手动安装

npm -v 在命令提示符查看npm版本

npm在安装node包是终端指令：npm install 模块名

npm在安装node包时有两种安装方式：本地安装 （local） 和全局安装（global）

node常用的一个模块 express 为例 安装

npm install express       本地安装
npm install express -g    全局安装

本地安装：将node包下载到当前目录（C:\Users\ASUA\Desktop\MyEngine）自动生成文件夹 mode_modules,本地安装的模块只能在当前目录所在的项目中使用
全局安装：会将node包安装到node根目录下（C:\Program Files\nodejs\node_modules），全局安装的模块可以在终端中使用模块语句

安装完成之后，可以执行更多的操作
npm ls 查看本地模块
npm ls -g查看全局模块
npm uninstall express 卸载模块
npm update express 更新模块
npm search express 搜索模块
npm publish express 发布模块

2・安装淘宝镜像：npm在安装模块时，受网速的影响较大，有时安装太慢，此时可以使用淘宝镜像cnpm替代npm下载安装模块

安装cnpm的指令：npm install -g cnpm --registry=https://registry.npm.taobao.org
用淘宝镜像安装express模块 

3・创建配置文件 package.json 作用是定义项目的属性信息，
创建方式：在终端中输入指令 npm init 初始化一个项目
设置的属性信息：
     name：项目名称（小括号中默认，和文件夹名相同，不能包含大写字母和汉字）
     version：项目的版本号（格式：a.b.c）
              版本号的命名：最原始的版本，其版本号一般用1.0.0，版本升级时会修改版本号，当有重大更新时，改a；当有新功能添加修改时，改b，修复漏洞/bug时，改c
	description：项目的描述信息
	entry point：入口文件
	test command：测试命令
	git repository：
	keywords：关键词
	author：作者
	license：许可证（默认ICS）
	Is this ok? (yes)是否确认
上述步骤执行完之后会在终端对应目录下生成一个package.json文件，json文件中有一个依赖项dependencies
依赖项是这个项目关联的模块，把模块关联到依赖项中之后，即使把模块删掉，也可以找回
通过npm install指令重新下载关联的依赖项（模块）
npm install express --save 安装模块时在后边添加-save，会把这个模块添加到依赖项中
4・在项目文件夹目录下新建一个文件index.js（这个文件夹要和配置文件中设置的entry point入口文件的文件名保持一致，即生成的配置文件中属性main的值）
文件中写服务器的相关代码，详情见文件

5、在工程目录下创建一个自定义文件夹public作为静态文件夹，在静态文件夹中创建静态文件index.html

6、启动服务器：在终端中输入指令 node index.js 运行服务器文件 index.js（注意：终端的目录应该是项目目录）

7、打开浏览器访问端口号为5000的本地服务器，在浏览器地址栏输入：localhost:5000 或者 127.0.0.1:5000
服务器会默认返回静态文件index.html给浏览器打开
注意：静态文件夹中必须包含一个名为index.html的文件

127.0.0.1:5000/1.jpg 获取1.jpg文件（访问路径中不能出现静态文件夹名字）
